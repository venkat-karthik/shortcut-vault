# Contributing to Shortcut Vault

First off, thank you for considering contributing to Shortcut Vault! It's people like you that make the workflow better for everyone.

## How to Contribute

### Adding New Shortcuts
If you have a shortcut that isn't in the vault:
1. Use the "Add New Shortcut" button in the app.
2. Or, if you want to contribute to the codebase, add it to `supabase/seed.sql` and open a PR.

### Reporting Bugs
- Use GitHub Issues to report bugs.
- Provide clear steps to reproduce the issue.

### Feature Requests
- We're always looking for ways to improve the vault. Feel free to open an issue with your idea!

## Style Guide
- Use descriptive names for "Action" (e.g., "Add Auto Layout").
- Use standard key names (Ctrl, Shift, Alt, Cmd).
- Keep descriptions concise but helpful.

## Pull Request Process
1. Ensure your code follows the existing style.
2. Update the `README.md` if your changes affect the setup or features.
3. Your PR will be reviewed as soon as possible!

Happy coding! 🚀
